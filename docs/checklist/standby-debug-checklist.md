# 待机功耗通用排查清单

## 适用场景

待机平均电流偏高、深睡占比低、周期性唤醒明显，或不同样机功耗差异大。

## 需要收集的信息

- 平台、软件版本、硬件版本。
- 测试场景和复现步骤。
- 期望电流、实测电流、电流曲线。
- kernel log、wakeup_sources、suspend_stats。
- 网络、外设、应用后台活动信息。

## 排查步骤

1. 确认测试条件是否与基准一致。
2. 判断系统是否进入目标低功耗状态。
3. 检查是否有长时间 active wakelock。
4. 检查是否有频繁 wakeup source。
5. 检查外设是否正确 suspend 或下电。
6. 检查应用、alarm、网络心跳和日志开关。
7. 对比正常样机和异常样机。
8. 修改后按相同条件复测。

## 初步分流

| 判断结果 | 下一步 |
|---|---|
| 无法 suspend | 查 wakelock、suspend blocker、后台服务 |
| 能 suspend 但电流高 | 查外设电源、GPIO、PMIC、硬件漏电 |
| 周期性唤醒 | 查 alarm、网络、RTC、中断 |
| 只有特定场景高 | 查对应模块，例如 Wi-Fi、BT、GNSS、Modem |
