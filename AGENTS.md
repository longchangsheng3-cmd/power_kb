# AGENTS.md

本项目是待机功耗分析与优化知识库，结合 Claude Code 和本地 RAG 使用。

## 工作原则

- 优先读取 `docs/` 下的知识内容，再给出分析建议。
- 分析问题时优先参考 `docs/checklist/` 和 `docs/cases/`。
- 对日志或现象不要直接下结论，先列出证据、假设和需要补充的信息。
- 输出建议时包含：可能原因、排查步骤、需要补充的信息、优化建议、验证方法。
- 形成可复用经验时，建议沉淀到 `docs/cases/` 或 `docs/checklist/`。
- 修改知识库内容时，保持 Markdown 结构清晰，优先使用表格、清单和小标题。

## 推荐分析流程

1. 理解用户输入的问题、平台、场景和功耗指标。
2. 使用 RAG 或全文搜索检索相关基础知识、检查清单和历史案例。
3. 阅读召回内容，提取与当前问题相关的证据。
4. 输出结构化分析报告。
5. 如有明确根因和验证结果，生成案例草稿。
6. 更新知识库后重新构建 RAG 索引。

## 报告要求

分析报告建议包含：

- 结论摘要
- 已知现象
- 关键证据
- 可能原因排序
- 排查步骤
- 优化建议
- 需要补充的信息
- 验证方法
